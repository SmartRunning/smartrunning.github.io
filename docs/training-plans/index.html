Training Plans

Training plans are personalized schedules that guide athletes through structured training programs. The SmartRunning platform automatically generates optimal training plans based on athlete performance data and training goals.

Plan Versioningâ€‹

Training plans support versioning, allowing you to track changes and maintain history of your training program adjustments.

Active Planâ€‹

  • Each athlete has ONE active training plan at any time
  • The active plan is the current plan being followed
  • Historical plan versions are preserved for reference
  • Switch between different plans using the "Set as Active" button

Version Historyâ€‹

Every training plan maintains a complete version history:

  • Version Numbers: Plans are numbered (v1, v2, v3, etc.)
  • Parent Tracking: Each version links to its previous version
  • Timestamps: Creation and regeneration dates are recorded
  • Reasons: Track why plans were regenerated (injury, illness, schedule changes)

Regenerating Plansâ€‹

When you need to adjust your training plan mid-program:

  1. Open Your Active Plan: View your current training plan
  2. Click "Regenerate": Starts the regeneration process
  3. Choose Settings:
    • From Date: Select the date to regenerate from (past sessions are preserved)
      • Minimum date: Plan start date + 1 day (ensures at least one baseline day)
      • Maximum date: Plan end date
      • Default: Later of today or plan start + 1 day
      • Important: At least one completed day is required as a baseline for regeneration
    • Reason: Choose why you're regenerating:
      • Illness/Injury - Recovery or health issue
      • Schedule Change - Modified training schedule or race date
      • High Workload - Training load feels excessive
      • Manual Adjustment - Coach-initiated change
      • Other - Different reason
  4. Adjust Parameters (Optional):
    • Training days per week
    • Training schedule (marathon date, blocks, taper weeks)
    • Current fitness state (CTL/ATL loaded from your current metrics)
    • Intensity levels (rTSS min/max)
    • All parameters from the original plan are pre-filled
  5. Reset to Original: Use this button if you've made changes and want to restore the original values
  6. Submit: The new version is generated while preserving completed sessions

What Gets Preserved:

  • All completed training sessions before the "from date"
  • Original training load metrics (CTL, ATL, TSB values)
  • Complete parameter history for comparison

What Gets Regenerated:

  • Future training sessions from the "from date" onward
  • Optimized workload progression based on current fitness state
  • Updated plan structure using your parameter changes

Viewing Version Historyâ€‹

  1. Open any training plan
  2. Click "View History" button
  3. View list of all versions showing:
    • Version number
    • Creation/regeneration date
    • Regeneration reason
    • Parameter changes (if any)
  4. Click "View" on any version to see historical data

Managing Multiple Plansâ€‹

You can generate multiple training plans for comparison:

  1. Generate Multiple Plans: Create different plans with various parameters
  2. Compare Plans: View each plan's structure and workload
  3. Set Active Plan: Choose which plan to follow by clicking "Set as Active"
  4. Confirmation Required: System asks for confirmation before switching active plans

Use Cases for Multiple Plans:

  • Compare different training volumes (3-day vs 4-day schedules)
  • Test different periodization strategies
  • Prepare backup plans for schedule changes
  • Plan for different race goals

Best Practicesâ€‹

  1. Regular Regeneration: Regenerate plans when recovering from illness, injury, or when workload feels excessive
  2. Document Reasons: Always select an appropriate regeneration reason for tracking
  3. Compare Versions: Review version history to understand how your training evolved
  4. Preserve History: Never delete old versions - they provide valuable training insights
  5. Plan Ahead: Generate alternative plans before major schedule changes

Technical Detailsâ€‹

Plan Structureâ€‹

Each training plan contains:

  • Individual training days with specific workouts
  • Progressive workload increases
  • Recovery periods
  • Taper phases
  • Performance predictions

Generation Parametersâ€‹

Plans are generated using:

  • Current fitness level (training load metrics)
  • Training schedule (days per week, total blocks)
  • Target race date
  • Personal performance data
  • Physiological parameters

All generation parameters are stored with each plan version, ensuring consistency when regenerating.

Data Preservationâ€‹

The platform ensures:

  • Historical versions are never modified
  • Completed sessions are preserved during regeneration
  • Training load continuity is maintained
  • Parameter history is tracked for analysis

Frequently Asked Questionsâ€‹

Q: Can I have multiple active plans? A: No, you can only have one active plan at a time to maintain training consistency. However, you can generate multiple plans and switch between them.

Q: What happens to my old plan when I regenerate? A: The old version is preserved as historical data. The new version becomes your active plan, and all version history is maintained.

Q: Can I undo a regeneration? A: Yes! Since old versions are preserved, you can view historical versions and set any previous version as active if needed.

Q: Do I lose my completed workouts when regenerating? A: No, all completed sessions before your selected "from date" are preserved. Only future sessions are regenerated.

Q: How do I know which parameters were used for each version? A: View the version history to see complete parameter details for each version, including any changes made during regeneration.

Q: Why can't I select certain dates when regenerating? A: The regeneration form validates date selection to ensure data integrity:

  • You must select a date at least one day after the plan starts (to have a baseline day)
  • You cannot select dates after the plan ends
  • For future-starting plans, the minimum date is automatically set to plan start + 1 day

Q: Why did my regeneration fail with a validation error? A: This usually happens due to data type mismatches. The system automatically converts numeric parameters to the correct format, but if you encounter this error, try refreshing the page and regenerating again. If the issue persists, contact support.
